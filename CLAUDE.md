# MusicVid - Christian Music Video Generator

## Project Overview
CLI tool that generates synchronized MP4 music videos from audio files using stock footage, beat-synced cuts, and whisper-based subtitles.

## Commands
- `python3 -m pytest tests/ -v` - run all tests (~790 tests)
- `python3 -m musicvid.musicvid song.mp3` - run the CLI (uses cache by default)
- `python3 -m musicvid.musicvid song.mp3 --new` - force recalculation, ignore cache
- `python3 -c "import musicvid; print(musicvid.__version__)"` - check version

## Architecture
4-stage pipeline: audio_analyzer â†’ director â†’ visual_router/stock_fetcher â†’ assembler, orchestrated by Click CLI in `musicvid/musicvid.py`.
- Audio analyzer: Whisper transcribe uses `language="pl"`, `initial_prompt` (Polish Christian vocabulary), `temperature=0.0`, `condition_on_previous_text=True`; default model is "small" (not "base"); segment filtering skips segments with stripped text < 2 chars; analysis dict includes `energy_peaks` key â€” `list[float]` of onset strength peak times computed via `librosa.onset.onset_strength` + `librosa.util.peak_pick` + `librosa.frames_to_time`; analysis dict also includes `energy_curve` â€” `list[[float, float]]` of `[time, energy]` pairs (RMS energy normalized 0.0-1.0) and `energy_mean` â€” `float` average normalized energy; `analyze_audio()` accepts `lyrics_path=None` â€” when provided, calls `align_lyrics` from `lyrics_aligner` internally (deferred import) so returned `lyrics` already contain file text with Whisper timing; saves to `audio_analysis.json` (not `analysis.json`)
- `--mode runway` (default): Stage 3 uses `VisualRouter` with BFL flux-dev image + Runway animate_image (image-to-video) + Pexels nature mix; director uses TYPE_VIDEO_RUNWAY for chorus/bridge, TYPE_VIDEO_STOCK for intro/verse/outro; caches to `image_manifest.json`; `--mode ai`: same VisualRouter path but director uses TYPE_AI (BFL Flux) + TYPE_ANIMATED; `--mode stock` uses `stock_fetcher` (Pexels API for all scenes)
- Hybrid visual sourcing: director outputs `visual_source` (TYPE_VIDEO_RUNWAY|TYPE_VIDEO_STOCK|TYPE_PHOTO_STOCK|TYPE_AI|TYPE_ANIMATED) and `search_query`/`motion_prompt` per scene; `VisualRouter` in `musicvid/pipeline/visual_router.py` dispatches to the correct API; `_validate_scene_plan` accepts `mode` param â€” defaults `visual_source="TYPE_VIDEO_RUNWAY"` in runway mode, `"TYPE_AI"` otherwise
- Stock content filtering: `sanitize_query(query)` in `visual_router.py` checks search queries against `BLOCKED_WORDS` (returns "BLOCKED") and `SAFE_QUERY_MAP` (returns safe alternative); `SAFE_QUERY_MAP` is ordered longest-first so "worship hands raised" matches before "worship"; called in `_route_video_stock` and `_route_photo_stock` before any API call; blocked queries skip stock APIs entirely and fall back to BFL; `BLOCKED_WORDS` covers non-Protestant religious terms (muslim, mosque, islamic, hindu, buddha, catholic, orthodox, etc.), alcohol (beer, wine, whiskey, drinking, bar, pub, nightclub, drunk), drugs/gambling (gambling, casino, cigarette, smoking, drugs, violence, fight, party, nightlife), and multi-word phrases (prayer rug, church interior, icon religious)
- Stock filtering tests: `tests/test_stock_filtering.py` â€” `TestSanitizeQueryBlocked`, `TestSanitizeQueryBlockedAlcohol`, `TestSanitizeQueryBlockedDrugsGambling`, `TestSanitizeQueryBlockedNewReligious`, `TestSanitizeQuerySafeReplacement`, `TestSanitizeQuerySafePassthrough`, `TestVideoStockSanitization`, `TestPhotoStockSanitization`, `TestDirectorPromptFiltering`
- VisualRouter fallback chain: TYPE_VIDEO_RUNWAY â†’ BFL flux-dev image + animate_image (RUNWAY_API_KEY required) â†’ Pexels stock (no BFL fallback); TYPE_VIDEO_STOCK â†’ sanitize_query â†’ simplified query (2 words) â†’ BFL (visual_prompt or "nature landscape peaceful"); TYPE_PHOTO_STOCK â†’ sanitize_query â†’ Unsplash â†’ Pexels video â†’ BFL; TYPE_ANIMATED no RUNWAY_API_KEY â†’ static BFL image (Ken Burns); TYPE_ANIMATED Runway failure â†’ static BFL image (Ken Burns)

## Caching
- Content-addressed cache in `output/tmp/{audio_hash}/` (MD5 of first 64KB, 12 chars)
- Cached files: `audio_analysis.json` (or `audio_analysis_{lyrics_hash}.json` when --lyrics used), `lyrics_aligned_{lyrics_hash}.json` (deterministic alignment result), `scene_plan.json`, `video_manifest.json`, `image_manifest.json` (ai mode)
- Clip mode adds separate cache files: `clip_{N}s.json` (clip selection), `scene_plan_clip_{N}s.json`, `video_manifest_clip_{N}s.json` / `image_manifest_clip_{N}s.json` â€” changing `--clip` only invalidates `clip_{N}s.json`
- Preset mode adds `social_clips_{N}s.json` (3-clip selection); cache key includes reel duration so `--reel-duration 30` creates a separate cache entry
- Stages 1-3 skip if cached; stage 3 also verifies video files exist on disk
- `--new` flag forces full recalculation (deletes cache dir)
- Cache utilities in `musicvid/pipeline/cache.py` (`get_audio_hash`, `load_cache`, `save_cache`)
